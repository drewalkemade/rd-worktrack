# Nodes: Export (Sage50 CSV / Summary CSV / DrewEdit XLSX)

**Canvas IDs:** `export_sage50`, `export_summary`, `export_drewedit`  
**Color:** Green  
**Panel:** `ExportPanel.jsx` (shared stub)  
**API:** TBD — `POST /api/periods/{id}/export/{type}`

---

## Purpose

Final output exports for the biweekly pay period. Three parallel export nodes each produce a different artifact from the reconciled and corrected data.

---

## Sage50 CSV (`export_sage50`)

Produces the employee payroll CSV for import into Sage 50 Accounting:

- One row per employee with biweekly totals: REG, OT1, OT2, per-diem count
- Employee names use the `sage50_name` alias (distinct from display name)
- Encoded as UTF-16 (required by Sage 50 import)
- File path follows the configured `SAGE50_EXPORT_PATH` — **do not change this path**
- Output matches the format expected by `PayrollChequeRun_v00.xlsm`

---

## Summary CSV (`export_summary`)

Human-readable summary of the pay period for internal filing:

- Per-employee breakdown: approved hours, timesheet hours, variances resolved, expenses
- Useful for audit trail and management review
- Format TBD

---

## DrewEdit XLSX (`export_drewedit`)

Bundle of corrected employee timesheet files:

- One `*_DrewEdit.xlsx` per employee who had `approved_wins` corrections
- Generated by the Modified Timesheets node upstream
- Exported/downloaded as a ZIP or individual files
- Audit trail: each file includes `generated_note` cells documenting every correction

---

## Business Rules

- All three exports require **Modified Timesheets** to be complete
- Sage 50 export must not be re-exported without incrementing a run counter or clearing the prior export
- `export_drewedit` files must never overwrite original source timesheets — the `_DrewEdit` suffix is enforced
- Non-per-diem expenses without receipts (`missing` status) must not appear in billable export outputs

---

## Inputs

- All feed from → **Modified Timesheets**
